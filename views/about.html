About FitLife Tracker

Final Project (Week 10). Production-ready web application with authentication, authorization, roles, and secure API endpoints.

What this application does

  • Domain: Fitness workouts tracking (realistic data)
  • Collections: users ↔ workouts (linked by workouts.userId)
  • CRUD: Create / Read / Update / Delete via Web UI
  • Pagination: Workouts list uses ?page=&limit=

Security & Access Control

  • Authentication: Sessions-based login (cookie sid)
  • Password security: bcrypt hashing
  • Roles: user and admin
  • Owner protection: user can update/delete only their own workouts
  • Protected endpoints: all write operations require login
